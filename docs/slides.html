A Project Manager
Simulation Environment

A deterministic, discrete-event week at a SaaS company. LLMs voice the coworkers but can never move the world — rules decide, LLM narrates — and every task carries a computable score band [no-PM baseline, OPT] so agents are graded purely on outcomes.

1 · world

One clock, one log, three kinds of change

  • Synchronous — agent tool calls mutate the world at that instant (every action costs 1 sim-minute).
  • Asynchronous — a min-heap of events: replies, wakeups, arrivals, belief updates. Time jumps, never crawls; inference latency never leaks into the sim.
  • Derived — task progress is a pure function of (config, clock, log): work "accrues overnight" with nothing running.
  • Event-sourced — state = fold(scenario, events); replay rebuilds any instant with zero LLM calls.
  • Causal — no undispatched event ever sits at/before the clock (property-fuzzed after every action).
  • Push-only — a DES has no polling: chat wakes the agent mid-advance, email arrives on a batch grid, the board must be read.
1 · world — communication

Channels are priced like real offices

channelspeedcostreal-life claim
chat reply in 5–45 min; wakes the PM instantly 20 focus-min tax on the receiver, serialized an interruption costs ~20 min of refocus; three pings = three broken flows
email read at the receiver's next batch (NPC wakeup / PM 30-min grid) zero tax async by social contract — nobody drops their editor for mail
meeting zero-latency exchange, live room, verbatim minutes blocks every attendee's calendar for the full slot 1 hour × 4 people = 4 person-hours — the most expensive artifact in a company

None of this is in any tool description. Agents learn it from evidence (reply timestamps, slipping completions), testimony (the tax is felt — people complain in persona), or reward (noisy trajectories simply score lower).

1 · world — tasks

Real work always has an owner

  • Arrivals land pre-assigned to a default holder — often the wrong one (the swamped volunteer). The announcement carries (ticket: id, currently on X).
  • The PM's levers: assign_task (move it) and reprioritize (reorder it — scheduling only, scoring weight stays authored).
  • One owner per task. Many hands = a meeting, never a longer assignee list.
  • Scheduler: deterministic, preemptive, work-conserving — everyone works their highest-priority ready task and switches the moment that changes; paused work resumes where it stopped.
  • No retroactive credit — work starts at max(arrival, assigned_at); acting early is worth something because acting late costs something.
  • The do-nothing baseline is a complete unmanaged org: defaults play out, badly. Score 0 = "added nothing beyond it."
2 · belief updates

Three layers of knowing

layerwho holds itbehavior
truthengine + evaluator only the scheduler's reality — nobody in the sim reads it, not even the task's holder
belief(t)each holder, about their own work honest but possibly wrong; matures on an authored schedule — Sarah believes 70% on Monday, realizes 55% on Tuesday, sometimes pings you proactively to confess
trackereveryone the record — lags, inherits stale beliefs; auto-updates only on completions, the one public reliable signal

Consequence: the interrogation hack is dead — asking everyone on Monday collects honest wrong answers. Authored red herrings (confident false org beliefs) price noise-resistance, and every update is a deterministic scheduled event: zero LLM on any information path.

2 · belief updates — one task's week

What the PM can know, and when

Mon 09:05 tracker: "~70% done, on track" belief: 70% truth: 55% Tue 14:00 belief_update fires -> holder realizes 55% "schema rework surfaced — ~18 focused hours left" (proactive ping) Wed–Thu asking again = testimony (persona-colored, still honest-but-wrong) Thu 16:30 task truly completes -> tracker flips to Done ← the reliable signal two completions ≈ a velocity estimate for that person

The skill being measured isn't extracting truth — there is none to extract — it's running a process: keep slack against noisy estimates, react fast to confessions, calibrate people from observed completions.

3 · skills

Hidden specialists: speed is a latent

npc: sarah.skills = {db: 1.3} dave.skills = {auth: 1.3, ops: 1.2} task: tags = ["db"] rate(person, task) = geomean of matching factors = 1.3 ← specialist works 30% faster
  • Frozen config, never a runtime LLM call — same physics for every rollout of a task (GRPO-safe), nothing promptable.
  • Hidden from the PM: not on the board, not in tool results. Learnable only from how fast work actually completes — routing the db task to Sarah is discovered, not documented.
  • The OPT ceiling routes by skill too (fastest eligible worker, clamped ≥1×), so the bound stays honest.
3 · skills — combined in meetings

The swarm: meetings can move work, boundedly

swarm_rate(task, attendees) = Σ contrib_i · wᵢ sorted desc w = 1, 0.6, 0.36, … diminishing capped at 2.5× Brooks's law meeting on task T banks deposit = rate × slot effort-minutes that LAND at meeting end
  • A task has one owner; putting more people on it is a meeting — a temporary, costed swarm.
  • Additive but diminishing: the second helper adds 60% of their rate, the third 36% — a war-room can ~double a bottleneck, never explode it (pure parallelism would break the OPT ceiling).
  • The trade is real: the swarm's deposit vs every attendee's blocked calendar. A good PM swarms the skill-matched people on the critical path — and nothing else.
4 · rubrics

Every task carries its own scale

< 0 made things worse
(taxes, churn, busywork)
0 = no-PM baseline
defaults play out, unmanaged
1 = OPT_ideal
frictionless exhaustive optimum
  • Both anchors are pure functions of the config — computable at authoring time, stamped into every scenario file.
  • score = (agent − baseline) / (OPT − baseline) makes 100 generated tasks commensurable; odds = N/(1−N) re-expands the top end.
  • Band width is authored difficulty: it equals how wrong the default ownership is — the generator tunes it, a gate rejects degenerate tasks.
4 · rubrics — the metrics

Outcomes only. No checks, no keywords, no judge.

COMPLETION = Σ w · (progress + α·done)/(1+α) did the work get done EFFICIENCY = Σ w · done · (horizon − done_at)/span how early it landed COMBINED = (completion + γ·efficiency)/(1+γ) the RL reward DONE-RATE = W(shipped)/W(all) what fraction shipped FAIRNESS = 1 − σ(hours worked / available) team health, per person

Noise punishes itself through physics: interruptions and meetings consume capacity → completions land later → the score drops. Measured: a chatty do-nothing PM scores below zero; reassigning work off the piled-up volunteer scores +0.41 → +0.91.

Anti-hacking by construction: made-up tasks carry zero weight · relabeling priority changes scheduling only · the tracker is never the truth · stakeholders can't be labor · unfair asks are rejected at authoring.

the pipeline

From one template to a graded curriculum

template ─→ generate (seeded physics remix) ─→ gate (valid · fair · band tier) ─→ stamp [baseline, OPT] ─→ probe ladder ─→ Pareto dashboard

  • Difficulty is measured, never asserted — fingerprint knobs predict it, model score-degradation confirms it.
  • Deterministic end to end: same seed + same actions = the same week, byte for byte. 40 tests, ~0.5 s, zero LLM calls.
  • Every run replayable and re-gradeable forever — trajectory, transcript, and scorecard are one click from the benchmark point.
